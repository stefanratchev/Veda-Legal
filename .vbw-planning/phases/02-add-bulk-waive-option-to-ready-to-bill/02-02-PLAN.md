## Objective

Add a "Waive" button to each client card on the Ready to Bill page. Clicking it opens a confirmation modal showing the client's unbilled hours and entry count (scoped to active date range). On confirm, call the bulk-waive API and refresh the list.

- @`app/src/components/billing/UnbilledClientsSection.tsx` -- main container, manages date range and search state (226 lines)
- @`app/src/components/billing/UnbilledClientCard.tsx` -- individual client card (178 lines)
- @`app/src/components/billing/WriteOffModal.tsx` -- reference for existing modal patterns in billing
- @`app/src/app/globals.css` -- design system CSS variables (--bg-deep, --bg-elevated, --bg-surface, --accent-pink, --danger)
- @`app/src/hooks/useClickOutside.ts` -- for modal close-on-outside-click
- @`app/src/test/utils.tsx` -- `renderWithProviders()` for component test setup
- The API contract is: `PATCH /api/timesheets/bulk-waive` with body `{ clientId: string, dateFrom?: string, dateTo?: string }` returning `{ success: true, updatedCount: number }`
- Use `fireEvent` from `@testing-library/react` (NOT `@testing-library/user-event` which is not installed)

### Task 3: Wire up modal and API call in UnbilledClientsSection

**File:** `app/src/components/billing/UnbilledClientsSection.tsx` (MODIFY)

1. **New state:**
   ```tsx
   const [waiveTarget, setWaiveTarget] = useState<{ clientId: string; clientName: string; totalHours: number; entryCount: number } | null>(null);
   const [isWaiving, setIsWaiving] = useState(false);
   ```

2. **Pass `onWaive` to each UnbilledClientCard:**
   ```tsx
   onWaive={(clientId, clientName, totalHours, entryCount) => setWaiveTarget({ clientId, clientName, totalHours, entryCount })}
   ```

3. **Render BulkWaiveConfirmModal:**
   - `isOpen={waiveTarget !== null}`
   - Pass `clientName`, `totalHours`, `entryCount` from `waiveTarget`
   - Pass current date range from filter state
   - `onClose={() => setWaiveTarget(null)}`

4. **`handleBulkWaive` function:**
   ```tsx
   const handleBulkWaive = async () => {
     if (!waiveTarget) return;
     setIsWaiving(true);
     try {
       const res = await fetch("/api/timesheets/bulk-waive", {
         method: "PATCH",
         headers: { "Content-Type": "application/json" },
         body: JSON.stringify({
           clientId: waiveTarget.clientId,
           ...(dateRange.from && { dateFrom: dateRange.from }),
           ...(dateRange.to && { dateTo: dateRange.to }),
         }),
       });
       if (!res.ok) throw new Error("Failed to waive entries");
       setWaiveTarget(null);
       // Refetch unbilled clients (use existing fetch mechanism)
     } catch (error) {
       console.error("Bulk waive error:", error);
     } finally {
       setIsWaiving(false);
     }
   };
   ```

5. **Refetch after waive:** Call the existing data-fetching function. Waived entries automatically disappear since unbilled-summary excludes `isWrittenOff = true`.

**Commit:** `feat(billing): wire bulk waive API call with confirmation modal`
