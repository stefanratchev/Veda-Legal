## Objective

Create the `PATCH /api/timesheets/bulk-waive` admin-only API endpoint that sets `isWrittenOff = true` on unbilled time entries for a given client, optionally scoped by date range. This is the backend foundation for the bulk waive feature on the Ready to Bill page.

## Tasks

### Task 1: Create bulk-waive API route handler

**File:** `app/src/app/api/timesheets/bulk-waive/route.ts` (NEW)

Create `PATCH` handler:

1. Auth: `requireAdmin(request)` -- return error if not ADMIN/PARTNER
2. Parse JSON body: `{ clientIds: string[], dateFrom?: string, dateTo?: string }`
   - `clientIds` is required, non-empty array of client UUIDs
   - `dateFrom`/`dateTo` are optional YYYY-MM-DD strings; if omitted, waive all unbilled entries
3. Validate inputs:
   - `clientIds` must be non-empty array of strings
   - If `dateFrom`/`dateTo` provided, validate they parse to valid dates and `dateFrom <= dateTo`
4. For each clientId, build Drizzle update query:
   - WHERE: `clientId = X AND isWrittenOff = false`
   - If date range: add `date >= dateFrom AND date <= dateTo`
   - Must also exclude entries linked to FINALIZED service descriptions (subquery on `serviceDescriptionLineItems` joined to `serviceDescriptionTopics` joined to `serviceDescriptions` where status = 'FINALIZED')
   - SET: `isWrittenOff = true, updatedAt = new Date().toISOString()`
5. Execute all updates (can use `Promise.all` for parallel per-client updates, or a single query with `inArray(timeEntries.clientId, clientIds)`)
6. Return: `{ success: true, updatedCount: N }`
7. Error handling: wrap in try/catch, return 500 with `errorResponse()` on DB errors

**Key design decision:** Accept `clientIds` as an array (not single clientId) so the frontend can batch all selected clients in one request instead of N parallel calls. This reduces network round-trips and makes the operation more atomic.
