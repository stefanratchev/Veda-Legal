Add optional `dateFrom` and `dateTo` query parameters to the GET handler in `/api/billing/unbilled-summary/route.ts`.

Parse from `request.nextUrl.searchParams`:
- `dateFrom` (string, "YYYY-MM-DD") — if present, add `gte(timeEntries.date, dateFrom)` to the WHERE clause
- `dateTo` (string, "YYYY-MM-DD") — if present, add `lte(timeEntries.date, dateTo)` to the WHERE clause (import `lte` from drizzle-orm)

These filters are ADDITIVE to the existing `gte(timeEntries.date, BILLING_START_DATE)` filter. When `dateFrom` is provided, use the LATER of `dateFrom` and `BILLING_START_DATE` (i.e., `dateFrom >= BILLING_START_DATE` is typical; if `dateFrom < BILLING_START_DATE`, `BILLING_START_DATE` still applies because of the existing filter). Since both are `gte` conditions and SQL evaluates them with AND, the existing `BILLING_START_DATE` filter naturally acts as a floor — no special logic needed, just add the new conditions alongside the existing ones.

No validation needed beyond basic string presence — the date columns handle comparison. The query structure (join chain, groupBy, orderBy) stays the same; only the WHERE clause gets additional optional conditions.

Keep the existing BILLING_START_DATE filter. Do NOT remove it.
