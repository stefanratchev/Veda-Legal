## Objective

Add a three-dot options menu to each client card on the Ready to Bill page with a "Write Off All" action. Clicking it opens a confirmation modal showing the client's unbilled hours and entry count (scoped to active date range). On confirm, call the bulk-waive API and refresh the list.

## Design Pattern: Three-dot Options Menu

The card gets a vertical ellipsis (three-dot) icon button in the top-right corner. It is near-invisible at rest and reveals on card hover or focus. The dropdown contains a single destructive "Write Off All" action, styled with `--danger` color. This matches the existing three-dot pattern on `LineItemRow` in the SD editor.

**Trigger button styling:**
- `absolute top-3 right-3` positioning within the card
- `opacity-0 group-hover:opacity-100 focus:opacity-100` — invisible until card hover (card wrapper needs `group` class)
- `text-white/30 hover:text-white/70 hover:bg-[var(--bg-surface)]` — ghost-quiet at rest
- `e.stopPropagation()` on click to prevent card-level click handlers

**Dropdown styling:**
- `bg-[var(--bg-surface)] border border-white/10 rounded-lg shadow-lg shadow-black/40`
- `animate-fade-up` per project animation rule
- Single menu item: "Write Off All" with small X icon, `text-[var(--danger)] hover:bg-[var(--danger)]/10`
- Close on outside click via `useClickOutside`

### Task 3: Wire up modal and API call in UnbilledClientsSection

**File:** `app/src/components/billing/UnbilledClientsSection.tsx` (MODIFY)

1. **New state:**
   ```tsx
   const [waiveTarget, setWaiveTarget] = useState<{ clientId: string; clientName: string; totalHours: number; entryCount: number } | null>(null);
   const [isWaiving, setIsWaiving] = useState(false);
   ```

2. **Pass `onWaive` to each UnbilledClientCard:**
   ```tsx
   onWaive={(clientId, clientName, totalHours, entryCount) => setWaiveTarget({ clientId, clientName, totalHours, entryCount })}
   ```

3. **Render BulkWaiveConfirmModal:**
   - `isOpen={waiveTarget !== null}`
   - Pass `clientName`, `totalHours`, `entryCount` from `waiveTarget`
   - Pass current date range from filter state
   - `onClose={() => setWaiveTarget(null)}`

4. **`handleBulkWaive` function:**
   ```tsx
   const handleBulkWaive = async () => {
     if (!waiveTarget) return;
     setIsWaiving(true);
     try {
       const res = await fetch("/api/timesheets/bulk-waive", {
         method: "PATCH",
         headers: { "Content-Type": "application/json" },
         body: JSON.stringify({
           clientId: waiveTarget.clientId,
           ...(dateRange.from && { dateFrom: dateRange.from }),
           ...(dateRange.to && { dateTo: dateRange.to }),
         }),
       });
       if (!res.ok) throw new Error("Failed to waive entries");
       setWaiveTarget(null);
       // Refetch unbilled clients (use existing fetch mechanism)
     } catch (error) {
       console.error("Bulk waive error:", error);
     } finally {
       setIsWaiving(false);
     }
   };
   ```

5. **Refetch after waive:** Call the existing data-fetching function. Waived entries automatically disappear since unbilled-summary excludes `isWrittenOff = true`.
