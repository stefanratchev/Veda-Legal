## Objective

Create the `PATCH /api/timesheets/bulk-waive` admin-only API endpoint that sets `isWrittenOff = true` on all unbilled time entries for a single client, optionally scoped by date range.

## Tasks

### Task 1: Create bulk-waive API route handler

**File:** `app/src/app/api/timesheets/bulk-waive/route.ts` (NEW)

Create `PATCH` handler:

1. Auth: `requireAdmin(request)` -- return error if not ADMIN/PARTNER
2. Parse JSON body: `{ clientId: string, dateFrom?: string, dateTo?: string }`
   - `clientId` is required (single client UUID)
   - `dateFrom`/`dateTo` are optional YYYY-MM-DD strings; if omitted, waive all unbilled entries for the client
3. Validate inputs:
   - `clientId` must be a non-empty string
   - If `dateFrom`/`dateTo` provided, validate they parse to valid dates and `dateFrom <= dateTo`
4. Build Drizzle update query:
   - WHERE: `clientId = X AND isWrittenOff = false`
   - If date range: add `AND date >= dateFrom AND date <= dateTo`
   - Exclude entries linked to FINALIZED service descriptions (subquery on `serviceDescriptionLineItems` joined to `serviceDescriptionTopics` joined to `serviceDescriptions` where status = 'FINALIZED')
   - SET: `isWrittenOff = true, updatedAt = new Date().toISOString()`
5. Return: `{ success: true, updatedCount: N }`
6. Error handling: wrap in try/catch, return 500 with `errorResponse()` on DB errors

Include validation inline (missing/empty clientId → 400, invalid date format → 400, dateFrom > dateTo → 400).

### Task 2: Write API route tests

Follow the canonical test pattern from `route.test.ts` references:

**Test cases:**

1. **Auth tests:**
   - Returns 401/403 when not authenticated (mock `requireAdmin` returning error)
   - Returns 401/403 when user is not ADMIN/PARTNER

2. **Validation tests:**

   - Returns 400 when `clientId` is empty string
   - Returns 400 when `dateFrom` is invalid format

3. **Happy path tests:**

Add test comments documenting: "unbilled-summary already filters `isWrittenOff = true`, so waived entries automatically disappear from the Ready to Bill view" and "POST /api/billing (SD creation) already filters isWrittenOff entries, so no changes needed there."
